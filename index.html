proof-agent

Independent adversarial verification for AI-generated code. Catches what workers miss before production.

0 Self-verification
5 Check points
3x Auto-retry

What It Does

  • Spawns independent verifier agent separate from the worker
  • Runs actual commands and provides evidence-based verification
  • Catches false claims, broken code, and security vulnerabilities
  • Validates factual claims in documentation and code comments
  • Provides audit trail with command output and evidence

Automatic Triggers

  • Subagent changes 3+ files in a single operation
  • Any file matching patterns: *auth*, *secret*, *permission*, Dockerfile, *.env*
  • User explicitly requests verification with command or flag
  • Critical system configuration files modified

Verification Checklist

  • Correctness: Does the code actually do what was requested?
  • Bugs & Edge Cases: Regressions, unhandled errors, missed cases
  • Security: Vulnerabilities, exposed secrets, permission issues
  • Build: Does it build, compile, or lint cleanly?
  • Facts: Are claims, version numbers, or URLs verifiable?

Quick Start

# Verify last commit
bash scripts/verify.sh

# Verify since main branch
bash scripts/verify.sh main

# Fact-check documentation
bash scripts/fact-check.sh *.md

Configuration

# proof-agent.yaml
thresholds:
  min_files_changed: 3
  always_verify:
    - "**/*auth*"
    - "**/Dockerfile"
retry:
  max_attempts: 3

Verification Outcomes

✓ PASS

All checks passed with evidence. Code is ready for deployment.

✗ FAIL

Issues found with specifics. Auto-retry up to 3 times if fixable.

△ PARTIAL

Some checks passed, others unverifiable. User decides next action.

How It Works

1
Trigger Detection
Monitors file changes and security-sensitive patterns
2
Spawn Verifier
Creates independent agent with no knowledge of worker results
3
Run Checks
Executes verification commands and gathers evidence
4
Assign Verdict
PASS, FAIL, or PARTIAL with supporting evidence
The worker and verifier must be separate agents. Self-verification is not verification.
— Core principle

⚠️ Experimental Tool

Proof Agent is an experimental verification system. While it catches many issues, you still need to understand your code and review results carefully. Use as a supplement to, not replacement for, human code review.
